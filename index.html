Merit Gem Example

Sep 11th, 2014 | Comments

Installation

  1. Gemfile에 gem 'merit', gem 'ambry 추가
  2. rails g merit:install 실행하여 마이그레이션 파일 생성
  3. rails g merit MODEL_NAME 실행 (예. user). 해당 모델에 has_merit 추가와 마이그레이션을 생성
  4. rake db:migrate 실행
  5. 뱃지를 정의하기 위해 config/initializers/merit.rb 설정
  6. 규칙을 설정을 위해 app/models/merit/*의 파일들을 수정

Badges

Creating Badges

config/initializers/merit.rb에서 뱃지를 생성합니다.

Merit::Badge.create! 배지 생성시 해시 값으로 이루어집니다:

  • :id integer (required)
  • :name this is how you reference the badge (required)
  • :level (optional)
  • :description (optional)
  • :custom_fields hash of anything else you want associated with the badge (optional)

Example

  • :image_url 현재는 더미이미지 주소 경로를 위해 추가한 옵션이며 해시로 접근합니다. (더미이미지에 한글 생성이 안됩니다 ㅜㅜ) 디자이너 분이 뱃지 이미지를 만들어 주시면 이미지 서버 경로로 변경하면 되겠습니다.

Defining Rules

뱃지는 자동적으로 어플리케이션안에서 규칙과 조건에 의해 생성됩니다. 옵션에 의해 영구적 또는 일시적으로 뱃지를 소유할 수 있습니다.

뱃지의 규칙 / 조건은 app/models/merit/badge_rules.rb안에 정의되어 있고 grant_on 블록 매개변수에 의해 호출 됩니다:

  • 'controller#action' a string similar to Rails routes
  • :badge corresponds to the :name of the badge
  • :level corresponds to the :level of the badge
  • :to the object’s field to give the badge to. It needs a variable named @model in the associated controller action, like @post for posts_controller.rb or @comment for comments_controller.rb.
    • Can be a method name, which called over the target object should retrieve the object to badge. If it’s :user for example, merit will internally call @model.user to find who to badge.
    • Can be :itself, in which case it badges the target object itself (@model).
    • Is :action_user by default, which means current_user.
  • :model_name define the controller’s name if it’s different from the model’s (e.g. RegistrationsController for the User model).
  • :multiple whether or not the badge may be granted multiple times. false by default.
  • :temporary whether or not the badge should be revoked if the condition no longer holds. false -badges are kept for ever- by default.
  • &block can be one of the following:
    • empty / not included: always grant the badge
    • a block which evaluates to boolean. It recieves the target object as parameter (e.g. @post if you’re working with a PostsController action).
    • a block with a hash composed of methods to run on the target object and expected method return values

Examples

Points

Defining Rules

포인트는 액션-트리거로 “meritable” 리소스에 전달합니다. app/models/merit/point_rules.rb에서 아래와 같이 규칙이 정의되어 있습니다:

score accepts:

  • score
    • Integer
    • Proc, or any object that accepts call which takes one argument, where the target_object is passed in and the return value is used as the score.
  • :on action as string or array of strings (similar to Rails routes)
  • :to method(s) to send to the target_object (who should be scored?)
  • :model_name (optional) to specify the model name if it cannot be guessed from the controller. (e.g. model_name: 'User' for RegistrationsController, or model_name: 'Comment' for Api::CommentsController)
  • :category (optional) to categorize earned points. default is used by default.
  • &block
    • empty (always scores)
    • a block which evaluates to boolean (recieves target object as parameter)

Rankings

배지와 같이 구체적인 액션에 전달되지 않아 cron 작업으로 랭크가 적용이 되었는지 여부를 테스트 하기 위해 정의해야 합니다.

Defining Rules

app/models/merit/rank_rules.rb에 정의되어 있습니다:

set_rank accepts:

  • :level ranking level (greater is better, Lexicographical order)
  • :to model or scope to check if new rankings apply
  • :level_name attribute name (default is empty and results in ‘level’ attribute, if set it’s appended like ‘level_#{level_name}’)

Uninstalling Merit

  1. Run rails d merit:install
  2. Run rails d merit MODEL_NAME (e.g. user)
  3. Run rails g merit:remove MODEL_NAME (e.g. user)
  4. Run rake db:migrate
  5. Remove merit from your Gemfile

RORLa 프로젝트 설정 사항

  1. 뱃지 규칙(badge_rules.rb 참고):
    • 포스트 작성 개수에 따라 뱃지를 받습니다.
    • 유저의 질문 개수에 따라 뱃지를 받습니다.
    • 유저의 답변 개수에 따라 뱃지를 받습니다.
  2. 포인트 규칙 (point_rules.rb 참고):
    • 포스트 작성시 포인트 10 증가
    • 질문 작성시 5 증가
    • 답변 작성시 5 증가
  3. 랭크(레벨) 규칙 (rank_rules.rb 참고):
    • 레벨 1 -> 유저의 point가 0이상 50이하 일때
    • 레벨 2 -> 유저의 point가 50초과 100이하 일때
    • 레벨 3 -> 유저의 point가 100초과 200이하 일때
  4. welcome 뷰에 유저 로그인시 보이는 Badge 레이아웃 추가(추후 유저 프로필 뷰로 이동 예정) (app/views/welcome/index.html.erb 참고)
    • 네비게이션 바에 레벨과 포인트 뷰 추가 (app/views/layouts/application.html.erb 수정)
  5. 각 컨트롤러의 액션 실행시 레벨 체크를 위한 helper 추가(상의 후 cron job으로 변경 예정) (merit_helper.rb 참고):
    • application_controller.rb
    • posts_controller.rb
    • questions_controller.rb
    • welcome_controller.rb

MyGame: name of your project

-p com.MyCompany.MyGame: package name for android

-p: which platform to run on. Options are ios,android,win32,mac and linux.

Setting

1). iTerm에서 SublimeText 열기

$ ln -s "/Applications/Sublime\ Text\ 2.app/Contetns/SharedSupport/bin/subl" ~/bin/subl

2). 터미널에서 파일명을 ⌘ + 클릭시 해당 앱으로 이동 가능

3). Preference Setting이 JSON으로 되어있어 손쉽게 설정 가능.

4). 단어 구분자 한 단어를 구분할 기호를 설정 CSS개발시 불편함 감소.

#io-footer {
	margin-top: 25px;
}

5). 마지막 빈 공백 삭제

"trim_automatic_white_space": true

6). EOF를 마지막 라인에 추가 true로 변경시 저장시 마지막 라인에 EOF글자가 추가 됩니다.

"ensure_newline_at_eof_on_save": false

7). 포커스 잃을 때 자동 저장 true로 변경시 탭을 이동할때 자동으로 저장됩니다.

"save_on_focus_lost": true

8). 들여쓰기 안내선

draw_normal은 indent에 맞게 세로선이 보이고 draw_active를 하면 현재 커서의 indent가 보입니다.

"indent_guide_options": ["draw_normal", "draw_active"]
